## Delivered

- Removed `endpoint_id` from the public `ugdr_qp_conn_info`; the record now contains only `uint32_t qp_num`.
- Added a daemon-global, nonzero QPN allocator that never reuses or wraps a number during one daemon process lifetime. A separate live-QPN index resolves QPs across Client sessions while the generation-safe typed registry continues to own QP lifetime.
- Added versioned fixed-width control payloads and Client adapters for QP query, modify, connection-info query, and atomic connect.
- Implemented consistent query snapshots, RESET to INIT, RESET/INIT/RTR/RTS to ERR, explicit SQD/SQE rejection, and exact-mask retry validation.
- Implemented same-daemon connect by live remote QPN. Success atomically commits only the local peer, retry fields, and RTS state; failures preserve both QPs and caller outputs.
- Removed live-QPN index entries on explicit destroy and session disconnect without recycling QPNs.
- Updated public contracts, libibverbs alignment notes, the atomic-connect decision, and the reviewed-source inventory.
- Added ABI, state/error matrix, cross-session lookup, destroyed-QPN, non-reuse, remote-unchanged, and independent-process Client/server coverage.
