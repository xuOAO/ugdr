## Delivered

- Added versioned, fixed-width `CREATE_QP` payload encoding for send/receive CQ identities, SQ/RQ WR capacities, SGE maxima, RC type, and `sq_sig_all`. Integer fields use network byte order; QP control messages carry no pointer, C/C++ struct, or file descriptor.
- Added a generation-safe typed QP registry and RESET-state QP metadata. Each QP belongs to one PD, owns internal SQ/RQ metadata, and requires its PD and both CQs to belong to one Context and Client session.
- Implemented strict QP lifecycle relations. Creation increments the PD child count once and each distinct CQ reference count once; a shared send/receive CQ is counted only once. Destruction removes the same relations symmetrically, while live QPs make PD/CQ destruction return `EBUSY`.
- Added Client QP proxies and activated `ugdr_create_qp` and `ugdr_destroy_qp` without changing `include/ugdr/api.hpp`. Invalid, stale, repeated, wrong-Context, and cross-epoch cases follow the reviewed `EINVAL` domain; creation preserves the caller's init attributes.
- Switched the daemon to the QP-capable control service while retaining the existing Device, Context, PD, MR, and CQ operations.
- Updated the current Client contracts to distinguish the implemented QP lifecycle subset from the still-unsupported QP state, connection, queue, WR, and WC operations.
- Added codec, relationship, stale/cross-session, malformed request, public API, and independent-process Client/server tests.
