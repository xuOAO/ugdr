## Delivered

- Added `tools/client-contracts.json` as the non-semantic machine inventory for the public header,
  `ugdr_api` target, all 19 public functions, the reviewed F02 feature/S01-S04 source revisions, and
  the complete Client contract set.
- Added `tools/check_client_contracts.py`. It rejects schema drift, unsafe or missing paths, CMake
  target loss, duplicate/unknown/missing symbols, stale or unreviewed sources, incomplete contract
  indexing, missing libibverbs-alignment coverage, and missing AGENTS.md routes without modifying the
  repository.
- Added the required negative fixture matrix for a missing contract, stale source revision,
  unindexed contract, absent AGENTS.md route, and omitted public symbol. Each invalid repository is
  rejected with a targeted diagnostic.
- Split the Client Harness into an independent C11 executable and a C++ API/placeholder executable.
  The C Client directly includes `ugdr/api.hpp`, uses the public MR key, SGE, WR/WC field paths, calls
  all 19 public functions, and links `ugdr_api` without a C++ wrapper.
- Added `ugdr_libibverbs_alignment`, which directly includes `<infiniband/verbs.h>` and compares the
  reviewed numeric constants and field types. MR, SGE, and Receive WR additionally require exact
  size, alignment, and per-field offsets; Send WR and WC compare only the reviewed relevant fields
  and offsets; the projected QP attribute record does not claim full `ibv_qp_attr` layout identity.
- Made a missing rdma-core development header a clear CMake configuration error. The validation
  target does not link or call libibverbs and requires no RDMA device, daemon session, or GPU.
- Integrated the checker and Harness into CMake, CTest's `ugdr_contract` label, `tools/ugdr lint`,
  the quality-command contract tests, AGENTS.md navigation, contract-source index, and required-path
  governance. The production dependency graph remains unchanged.
- Kept every runtime entry point as an explicit unsupported placeholder. Pointer results, `errno`,
  direct/negative integer error domains, caller sentinels, inputs, output records, WR ownership, and
  WC non-production are checked independently; no fake handle, state, consumption, or completion was
  introduced.
