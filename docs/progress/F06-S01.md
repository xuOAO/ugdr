## Delivered

- Changed the internal Local Transport request contract from one whole-WR request with a segment vector to one continuous payload per request. Each payload carries its parent identity, logical byte offset, index/count, continuous source daemon address, and parent remote base.
- Added a default `8192`-byte, internally configurable requester split cursor. It validates and resolves the complete WR before transmission, never crosses an SGE boundary, retains the current payload under request-transport backpressure, and releases the Send WQE only after every declared payload enters the transport.
- Changed `CopyBackend` to accept and complete one payload task at a time. The scripted backend can complete arbitrary accepted positions for deterministic out-of-order tests; the existing Mock GPU backend now executes one device-to-device copy per payload.
- Added responder-side parent aggregates with one full-range remote MR validation, one Receive WR consumption for Write With Immediate, terminal bitmap/result storage, deterministic lowest-index payload failure selection, and an ordered parent-response queue. Backend, response-transport, and Receive CQ backpressure retain ownership without dropping or duplicating parent results.
- Added an optional internal parent completion observer on the requester. It observes all successfully consumed parent Responses, including unsignaled WRs, without changing the public API or using Send WC count as a throughput proxy.
- Added `ugdr_loop_worker_payload_benchmark`, a control-only full-loop harness using fake MR metadata and an address-opaque bounded backend. Its measured path includes `post_send_chain`, SQ, requester/responder Loop Workers, Local Transport, per-payload completion, parent aggregation, WR-level Response, and observer consumption.
- The benchmark emits the complete case parameters plus completed parent/task counts, logical bytes, parent MWR/s, payload MTask/s, logical payload GB/s, and WR P50/P99. It contains no CUDA Runtime call, `cudaMemcpy`, real GPU buffer access, or `MockGpuBackend`, and its executable has no CUDA runtime/driver link dependency.
- Extended deterministic tests for WRs below, equal to, and above the payload limit; non-divisible tails; multi-SGE boundaries; out-of-order payload completion; lowest-index failure selection; signaled, unsignaled, and `sq_sig_all`; Write With Immediate; and request/backend/response/CQ backpressure. The CUDA integration test now grants one explicit backend progress token per payload.
