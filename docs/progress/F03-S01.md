## Delivered

- Added the business-neutral `ugdr_ipc` target with a fixed-width, big-endian Unix Domain
  `SOCK_SEQPACKET` envelope, bounded inline payload, explicit version/flags/request/status fields,
  and `sendmsg`/`recvmsg` `SCM_RIGHTS` attachments. C/C++ object memory, process pointers, and raw
  sender fd numbers are never serialized.
- Added move-only `UniqueFd` and message-level fd ownership. Invalid version, flags, payload length,
  fd count, truncation, or ancillary shape is rejected before handler dispatch; every received fd is
  first placed under RAII so malformed messages cannot leak an unclaimed descriptor.
- Added synchronous `IpcClient` request/response correlation and `IpcServer` listener, session map,
  one-thread `poll()` dispatch, and exactly-once disconnect notification. The server refuses to
  replace an existing socket path and removes its path only when device/inode still identify the
  socket it created.
- Added the typed UGDR adapter. Object identity, length, access, opaque bytes, and fd attachment
  indices use explicit versioned fields; request and response decode validate the complete payload,
  attachment count, and every index. The control service receives ownership of decoded fds and may
  return owned response fds without exposing them to the generic IPC layer.
- Added `ControlIpcHandler` and an S01 unsupported service that compose the generic server and typed
  adapter without creating Context, PD, MR, CQ, QP, SQ, or RQ resources. `ugdr_daemon --socket PATH`
  runs this composition in a single control thread; its signal path stops and closes all sessions.
- Added protocol and adapter unit tests plus an independent-process client/server integration test.
  Coverage includes zero, one, and multiple fds, sender ownership, usable received descriptors,
  request correlation, disconnect, invalid flags, truncated packets, illegal payload shapes, and
  illegal fd indices.
- Registered the new production target, tests, required paths, and transitive CMake dependencies in
  the module-boundary policy and synchronized repository skeleton.

## Deviations and Remaining

- No reviewed design deviation was introduced. S01 deliberately returns `EOPNOTSUPP` from the
  composed control service and does not create or open any verbs or CUDA resource.
- The existing no-argument daemon smoke path remains finite for repository smoke testing;
  `ugdr_daemon --socket PATH` is the S01 control-server runtime path. Production socket location,
  daemonization, service management, and resource handlers belong to later F03 steps.
- `SCM_RIGHTS` transport proves fd transfer and ownership, but CP/SQ/RQ allocation and interpretation
  are outside S01. The adapter carries opaque bytes and validated fd indices so those later handlers
  do not require a second IPC protocol.
