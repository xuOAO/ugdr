## Delivered

- Replaced the successful F01 `api_placeholder` with C-linkage `ugdr_*` declarations for device/context, PD, MR, CQ, RC QP, connection exchange, WR posting, and WC polling.
- Added opaque public object and record types. Records owned by F02-S02 through F02-S04 remain incomplete so F02-S01 does not freeze lifecycle, connection, WR, or WC layouts prematurely.
- Added the reviewed v1 enum subset for RC QP type and state, RDMA Write opcodes, signaling, completion status/opcodes, and MR access. Published values align with the installed libibverbs primary headers.
- Added linkable placeholders for every public function. Pointer-returning functions fail with null plus `errno=EOPNOTSUPP`; direct errno-domain functions return `EOPNOTSUPP`; `ugdr_close_device` returns `-1` plus `errno`; `ugdr_poll_cq` returns `-EOPNOTSUPP`. No placeholder allocates state, writes outputs, consumes WRs, or creates WCs.
- Added `docs/contracts/README.md`, `public-api.md`, and `libibverbs-alignment.md`, and registered the contract directory in `AGENTS.md`, documentation governance, module-boundary policy, and the generated repository skeleton.
- Added `ugdr_api_contract`, covering function signatures, enum values, explicit failure behavior, unchanged output parameters and `bad_wr`, and alignment-matrix coverage. Updated the minimal Client executable to exercise the supported placeholder failure contract.
