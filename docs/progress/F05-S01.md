## Delivered

- Added internal `DatagramOpcode`, `DatagramResult`, `ResolvedSourceSegment`, `RequestDatagram`, and `ResponseDatagram` value types under `src/worker/`, with only the reviewed logical fields and result categories.
- Added a passive `LocalTransport` with independent request and response capacities. Each direction uses an ordinary `std::deque`; the implementation creates no thread, exposes no progress method, and provides no concurrency guarantee.
- Implemented non-blocking request/response push and pop. Full push and empty pop return `false`; failed push leaves queue state unchanged, failed pop preserves the caller output, successful pop immediately restores capacity, and the two directions do not affect each other.
- Added the `ugdr_local_transport_test` executable and `ugdr_local_transport` CTest for request/response field round trips, every response result, FIFO ordering, empty/full behavior, capacity recovery, direction independence, and failure side effects.
- Kept serialization, wire format, Worker integration, in-flight state, Mock GPU behavior, WC generation, SPSC synchronization, atomics, and mutexes outside this scope.
