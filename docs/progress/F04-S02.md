## Delivered

- Implemented Client-side Send/Receive posting helpers that validate a linked WR chain, copy scalar fields and SGEs directly into shared SQ/RQ slots, consume both wrap-around batch spans in logical order, and publish once per accepted batch.
- Restricted Send posting to RTS and Receive posting to INIT/RTR/RTS. Immediate structural failures return `EINVAL`; full queues return `ENOMEM`; both identify the first unaccepted WR through `bad_wr` while retaining an accepted prefix.
- Preserved the verbs boundary between posting and execution: lkey, rkey, address, range, and access checks are deferred to the worker. A normal Write stores zero immediate data; Write With Immediate preserves the supplied value.
- Added one posting mutex and cached state per QP. The mutex covers a whole linked chain and coordinates modify, query, connect, destroy, and stale cleanup; posting does not hold the control Client mutex or issue IPC. Different QPs may post concurrently.
- Kept WR and SGE pointers out of shared memory. The steady-state helper performs no heap allocation, syscall, descriptor-pool lookup, or control-plane request.
- Updated the public API, WR/WC, and libibverbs-alignment contracts plus the reviewed-source inventory. `ugdr_poll_cq` remains the only public data-path placeholder in this scope.
- Added unit coverage for both opcodes, signaling, zero/multiple SGEs, copy lifetime, malformed input, successful prefixes, full queues, wrap-around, two-span batches, and allocation counting.
- Added a Client/server test that posts through the public API while the daemon consumes independent mappings, verifies state gates and unchanged control request counts, and checks every concurrent two-WR chain is contiguous and observed exactly once.
