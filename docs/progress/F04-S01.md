## Delivered

- Added `ugdr_queue` with a versioned, page-aligned shared SPSC ring; fixed-width header fields; separate head/tail cache lines; checked sizing; `memfd_create`; sealing; `MAP_SHARED`; and RAII cleanup.
- Added zero-copy batch reserve/publish and peek/release. A batch exposes up to two slot spans across wrap-around and performs one shared cursor publication for the committed count.
- Producer and consumer keep cache-line-separated local cursors plus shadow copies of the remote cursor. The remote cursor is refreshed only when cached space/data cannot satisfy the request; steady-state slot addressing advances a local index without division.
- Because each SPSC side is the sole writer of its cursor, publication uses release-store rather than an atomic read-modify-write; the opposite side uses acquire-load before accessing or reusing slots.
- Added direct SQ, RQ, and CQ descriptor layouts. Queue capacities remain exact, including non-power-of-two values; no descriptor pool, freelist, process pointer, mutex, or C++ container is stored in shared memory.
- CQ creation now returns one completion-ring fd. QP creation returns SQ then RQ fds. Versioned network-order control descriptors bind each fd to its kind, capacity, and stride.
- Client CQ/QP creation validates identity, descriptor payload, fd count/order, `fstat` size, and shared header before publishing a proxy. Mapping failures issue compensating destroy requests.
- Daemon records retain the owner mappings until successful destroy or session cleanup. Client mappings are released only after successful public destroy or stale connection detection.
- Public `ugdr_post_send`, `ugdr_post_recv`, and `ugdr_poll_cq` remain unsupported as required by S01.
